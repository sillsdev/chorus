Chorus will allow you to share files between team members. In this version, only sharing via USB keys is enabled.

Step 1: Begin by having one team member start a repository which holds the files you want to share.  If your application (e.g. FLEx or WeSay) is Chorus-ready, open your project and give the "Sync" command.  This will create a repository and check your work into it.

Step 2: Prepare a USB Key for backup & sharing.

 

Now that you have a reposistory on a USB key, you can always plug it in and quickly sync with it.  This will not only backup your work to the key, but will do so in a way that preserves the entire history of your work since you started using Chorus.  You can prepare as many keys as you like and rotate them however you like, for multiple backups.  Make sure to keep one or more of the keys away from the computer, in case of fire, flood, or theft.

Step 3: Take a USB key which holds a repository to the computers of other team members, run Chorus, and copy the repository.

Step 5: If this repository you have open now should be protected as a "master"/"gold" version of the database,  indicate that in the settings panel.

 This will cause Chorus to share changes made on the master with the rest of the team, but the master will not pull in work from anyone else.  Eventually, your applications (e.g. FLEx) will make it easy to view proposed changes from the team and selectively bring them into the master.

Sharing Work

On a regular basis, do a Sync to a USB Key and take/send it to your other team members. Have each one sync with it. Each member will get all the work, of all the members, that the key has seen. Where there are conflicts, this version of Chorus will choose to keep the incoming work at the expense of the existing work.  The conflict will show up in the log, and will eventually be easy to reconsider from within your application (e.g. FLEx or WeSay). 

In the near furture, you will be able to register the locations of non-USB repositories here.  These will include paths over a Local Area Network for teammembers in the same office, and an internet address for a Chorus-compatible server, for team members located far from each other.
